# 剧本转分镜

## Workflow

优先使用 `script2storyboard` CLI 生成 `文字分镜.md`：

```bash
script2storyboard convert <剧本文件> --output 单集制作/EP001/文字分镜.md
script2storyboard convert <剧本文件> --audience-mode female --output 单集制作/EP001/文字分镜.md
script2storyboard convert <剧本文件> --audience-mode male --max-shot-seconds 3 --output 单集制作/EP001/文字分镜.md
```

如果用户要求人工撰写、CLI 未安装，或当前环境无法调用模型，则按下方规则直接输出 Markdown。

## Output Rules

- 输出文件默认命名为 `文字分镜.md`。
- 每个 Clip 默认 15 秒，标题使用 `## Clip N`。
- 文件开头必须包含 `## 参考图分配`，明确图片、音频、视频编号的用途。
- 文件开头必须包含 `## 连贯性设定`，明确人物站位、场景轴线、正反打方向、主要转场锚点和景别节奏。
- 每个 Clip 包含 `### 基础信息` 和 `### 分镜列表`。
- 每个 Clip 必须包含 `### 导演调度`。
- 基础信息至少包含人物、场景、时长、内容概述。
- 分镜条目使用 `0-3s:` 这类时间段，时间段合计覆盖当前 Clip。
- 分镜条目建议写成 `- 0-3s: [中近景/轴线左侧/承接上镜] ...`。

## Writing Rules

- 保留剧本中的每一句台词原文，不用动作概括替代台词。
- 每次提及角色都写成 `角色名@图片N`，不要只写角色名。
- 有参考图的道具写成 `道具名@图片N`。
- 用具体物理动作、表情、空间关系、镜头运动、光线方向和色温表达情绪。
- 同一空间内以人物/物体连线为轴线，镜头保持同一180度侧；需要换侧时用空镜、正面中性镜头或空间转场重建轴线。
- 识别人物正反打和场景正反打，保持视线方向、画面左/右关系、背景方位连续。
- 不乱切：只有台词、动作、视线、空间信息、道具状态或情绪节奏变化时才切镜。
- 单镜不要过长；用中景、中近景、近景、特写、空镜、侧面角度和道具细节消解视觉疲劳。
- 转场要有锚点，可用手机、脸、门、血滴、道具、空间入口或灯光变化承接。
- 景别要连贯，都市情感戏避免无动因大跳切；打斗、危机、强爽点可有更大跳变但要写明动因。
- 不写视频提示词，不写视频生成命令，不输出 JSON。

## Audience Modes

- `general`：平衡叙事清晰、人物关系、空间连续和镜头节奏。
- `female`：强化情绪流、关系张力、凝视/反应、细微身体动作和氛围转场。
- `male`：强化目标推进、爽点、压迫感、力量变化、空间调度和冲击空镜。

## CLI Notes

- 配置项：`S2S_ARK_API_KEY`、`S2S_TEXT_MODEL`、可选 `S2S_ARK_BASE_URL`。
- 运行 `script2storyboard auth check` 检查配置。
- 运行 `script2storyboard check <文字分镜.md> --max-shot-seconds 4` 离线检查结构和关键规则。
- `check` 将台词遗漏、Clip 编号等作为错误；导演调度缺失作为 warning。
